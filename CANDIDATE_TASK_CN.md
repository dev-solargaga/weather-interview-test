# 候选人任务：天气应用

你接手了一个使用 Vue 和本地后端开发的小型天气应用。

应用目前已经可以根据城市名称查询和显示当前天气。

## 时间

你将有：

* 正式考核前 15 分钟安装依赖并运行项目
* 60 分钟完成编程任务

这不是 UI 设计考核。界面简单、清楚并且可以正常使用即可。

## 你的任务

请选择其中一条路线。

### 路线 A

完成：

* 1 个前端任务
* 2 个后端任务

### 路线 B

完成：

* 1 个全栈任务
* 1 个后端任务

无需完成所选路线以外的任务。

修改后，现有的城市天气查询功能必须继续正常运行。

---

# 前端任务

选择一个。

## FE1：验证与用户反馈

* 删除城市名称前后的空格
* 阻止空白查询
* 显示加载和错误状态
* 加载时避免重复提交
* 区分“城市不存在”和“后端连接失败”

## FE2：防抖查询

* 用户停止输入后自动查询
* 不查询空白内容
* 避免不必要的请求
* 防止旧请求覆盖较新的查询结果

## FE3：最近查询记录

* 使用浏览器存储保存成功的查询
* 最多保留五个城市
* 不区分大小写地去除重复记录
* 允许用户再次查询历史城市
* 存储数据错误时页面仍可正常运行

---

# 后端任务

路线 A 选择两个，路线 B 选择一个。

## BE1：验证与错误处理

* 删除城市参数前后的空格并进行验证
* 拒绝空白输入
* 城市不存在时返回合适的响应
* 使用统一的错误格式
* 不向前端返回原始异常信息

## BE2：第三方 API 可靠性

* 为 Open-Meteo 请求设置超时时间
* 处理连接失败和请求超时
* 处理缺失或错误的 API 数据
* 添加有用的后端日志
* 向前端返回安全的错误信息

## BE3：内存缓存

* 缓存成功的天气查询结果
* 使用标准化后的城市名称作为缓存键
* 设置可配置的过期时间
* 不缓存失败的请求
* 展示缓存命中和缓存未命中的情况

---

# 全栈任务

路线 B 选择一个。

前端只能调用本地后端。

## FS1：空气质量

使用 Open-Meteo Air Quality API 显示：

* AQI
* PM2.5
* PM10

使用后端已经取得的城市经纬度。

如果空气质量数据无法获取，原有天气结果仍然必须可以显示。

使用简单的区域或卡片展示即可。

## FS2：多个同名城市

部分城市名称可能对应多个地点。

不要永远使用第一个查询结果：

* 后端最多返回五个匹配地点；
* 前端使用简单列表或下拉选项显示；
* 显示城市、地区和国家；
* 允许用户选择一个地点；
* 查询所选地点的天气。
