# 候选人任务：天气应用

这是一份简短的现场编程考核，涵盖 Vue 前端和本地后端。

## 时间安排

- 考核开始前 15 分钟：安装依赖并启动项目
- 60 分钟：完成编程任务
- 不要求进行大型 UI 重构；界面只需保持简单、清晰且可用
- 本次考核主要评估后端能力、AI 集成、数据验证和系统可靠性
- 你可以根据完成必做全栈任务的需要修改前端

## 必做任务

请完成：

1. **FS：AI 天气建议**
2. **一个后端任务**

如果仍有剩余时间，你可以完成一个前端任务，或一个可选的全栈扩展任务。

重要：现有的城市天气查询功能必须继续正常工作。

---

## 必做全栈任务

### FS：AI 天气建议

项目已在所分配的后端实现中提供一个确定性的模拟 AI 天气服务：

- Python/FastAPI
- Java/Spring Boot

该模拟服务接收真实的当前天气数据，并可能返回以下字段：

- `summary`
- `recommendation`
- `riskLevel`

有效响应应包含：

- `summary`：对当前天气情况的简短描述
- `recommendation`：提供给用户的实用建议
- `riskLevel`：`low`、`medium` 或 `high`

#### 后端要求

- 使用真实的当前天气数据调用已提供的模拟 AI 服务
- 将所有 AI 生成的输出视为不可信数据
- 验证 `summary` 是否为非空字符串
- 验证 `recommendation` 是否为非空字符串
- `riskLevel` 只允许为 `low`、`medium` 或 `high`
- 处理格式错误的输出
- 处理字段缺失
- 处理无效的风险等级
- 处理服务失败、超时或异常
- 返回安全的回退结果，并确保正常天气响应不会因此失败
- 确保 AI 生成内容不会替换或修改真实天气数据

#### 前端要求

在一个简单区域中显示：

- 天气摘要
- 建议
- 风险等级

如果 AI 建议无效或不可用，真实天气结果仍必须继续显示。

#### 必须验证的情况

至少演示以下情况：

- 有效的 AI 输出
- 字段缺失
- 无效的风险等级
- 格式错误的输出
- 服务异常或超时

不要使用真实的 AI API 或模型替换已提供的模拟服务。

不要引入：

- 真实的 OpenAI 依赖
- API 密钥
- 外部 AI 服务

## 后端任务

选择一个完成。

### BE1：验证与错误处理

- 验证并规范化城市输入
- 去除不必要的前后空格
- 拒绝空输入
- 使用合适的 HTTP 状态码
- 返回统一格式的错误响应
- 不向前端暴露原始异常信息

### BE2：外部 API 可靠性

- 为 Open-Meteo 请求添加明确的超时设置
- 处理连接失败和请求超时
- 处理上游数据缺失或格式错误
- 添加有用的后端日志
- 向前端返回安全且容易理解的错误信息

### BE3：内存缓存

- 缓存成功的天气查询结果
- 规范化缓存键
- 使用可配置的过期时间
- 不缓存失败请求
- 能够演示缓存命中和缓存未命中
- 在适用的情况下考虑线程安全或并发请求

## 可选前端任务

除完成 FS 所需的前端修改外，以下前端任务均为可选。

如果仍有剩余时间，请选择一个完成。

### FE1：验证与反馈

- 去除城市输入前后的空格
- 阻止空搜索
- 显示加载状态和错误状态
- 请求进行中时防止重复提交
- 区分“城市未找到”和“后端服务失败”

### FE2：防抖搜索

- 在用户停止输入后再发起搜索
- 避免空请求和不必要的请求
- 防止较早的响应覆盖较新的搜索结果
- 取消过期请求，或安全地忽略过期响应

### FE3：最近搜索记录

- 使用浏览器存储
- 只保存成功的搜索
- 最多保留五条记录
- 删除不区分大小写的重复项
- 允许用户再次搜索历史城市
- 安全处理缺失或损坏的存储数据

## AI 辅助开发要求

允许使用 AI 工具。

在要求 AI 修改代码之前，请先说明：

- 你对问题的理解
- 你提出的解决方案
- 你预计需要修改的文件或逻辑
- 你计划验证的风险和边界情况

使用 AI 后，你仍需负责：

- 检查所有生成的修改
- 拒绝或修正不合适的建议
- 运行应用
- 测试成功和失败场景
- 验证现有功能仍然正常工作
- 理解并能够解释最终实现

面试过程中，我们可能会要求你展示或总结你是如何使用 AI 的。

我们会根据实际情况公平评估 AI 的使用。候选人不会仅仅因为使用 AI 而被扣分。我们将评估 AI 是否被合理使用，以及候选人是否理解、测试、验证并能够解释最终代码。
