# Candidate Task: Weather Search Application

You have inherited a small weather application. It works for common cases, but it was completed quickly and has room for improvement.

## Your task

1. Run the application locally.
2. Inspect both the frontend and backend.
3. Identify technical, usability, reliability, or maintainability issues.
4. Implement the improvements you consider most important.
5. Be prepared to explain:
   - what you changed;
   - why you prioritised those changes;
   - what you would do next with more time;
   - how you verified your work.

## Minimum expected behaviour

- A user can search for a city.
- The frontend calls the local backend, not Open-Meteo directly.
- The backend retrieves location and current-weather data from Open-Meteo.
- The page displays the resolved city, temperature, apparent temperature, wind speed, and weather code.

## Optional extensions

Choose any that demonstrate your strengths:

- better validation and error handling;
- loading and empty states;
- automated tests;
- cleaner architecture;
- request cancellation or debouncing;
- caching;
- accessibility improvements;
- Docker support;
- API documentation;
- unit conversion;
- forecast display.

Do not rebuild the entire project unless you can justify why that is the best use of the interview time.

### Use of AI tools is permitted. However, before asking AI to make any changes, you must first explain your understanding of the problem and describe the solution you would propose. After using AI, you are also responsible for reviewing, testing, and verifying the updated code so that you can clearly explain the changes during the follow-up discussion.

### The use of AI tools will be assessed fairly and in context. Candidates will not be penalized simply for using AI; however, we will evaluate how appropriately it was used, whether the candidate understood the proposed solution, and whether they were able to review, test, verify, and explain the final code.
